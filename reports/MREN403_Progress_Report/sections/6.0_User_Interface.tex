Progress for the basic web interface (\textbf{FLT-UI-001}) has been steadily made. The key achievements include ROS2 integration, 3D visualizations, and diagnostics. Using WebSockets for communication, the web dashboard can read ROS2 topics and render the meshes defined in the URDF. This capability enables the web interface to replicate core RViz utilities, such as setting an initial pose and generating navigation waypoints, including manual selection (see \hyperref[sec:autonomy]{\textit{Autonomy Subsystem}} for detailed information on waypoint generation). Using WASD, the web dashboard can also override the Nav2 controls. The dashboard features a 3D visualization environment that displays the robotâ€™s current pose, costmaps, Nav2 paths, etc. (see \autoref{fig:web_dashboard} for a screenshot of the dashboard interface). A significant technical challenge was correctly interpreting the Transform (TF) tree, particularly for the LiDAR visualization. The web dashboard also includes various diagnostics to ensure the required topics and services are active.
